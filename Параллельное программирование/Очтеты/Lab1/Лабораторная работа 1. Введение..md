В работе используется компилятор GCC (gcc14). Для компиляции с агресивной оптимизацией используется аргумент командной строки `-O2`, для компиляции без оптимизаций компилятора - `-O0`. Для автоматизации сборки и очистки используется утилита `Make`.

## Вывод
В ходе работы был сделан вывод о том, что построчная обработка матриц происходит быстрее, чем постолбцовая. Это проистекает из особенностей работы процессора с наборами данных из памяти. Также сделан вывод о том, что агрессивные оптимизации компилятора (в том числе векторизация) значительно ускоряют работу с матрицами. При этом заполнение матрицы рандомными значениями происходит гораздо медленне ввиду накладных расходов на генерацию псевдослучайных чисел.
