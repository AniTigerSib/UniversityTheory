**Параллелльное программирование** - это способ организации компьютерных вычислений, при котором программы разрабатываются как набор взаимодействующих вычислительных процессов, выполняемых одновременно.
- Одновременность процессов обеспечивается на физическом уровне.
- Используется потенциал многоядерных и многопроцессорных систем.
- Целью является повышение производительности вычислений.

**Многозадачность** - это способность системы олновременно выполнять несколько независимых вычислительных процессов (задач).
- Одновременность процессов может быть обеспечена как на физическом, так и на логическом уровне - временное разделение ресурса процессора.
- Целью является упрощение создания сложных программных систем, повышение универсальности вычислительных систем, повышение надежности и устойчивости.

#### Виды многозадачности
- Кооперативная
- Вытесняющая
- Истинная

#### Виды задач
- Процесс - характеризуется контекстом, включающим распределение памяти и используемые ресурсы системы, и содержит не менее 1 потока.
- Поток - последовательность инструкций, выполняемая в контексте потока.

#### Повышение производительности вычислений
- Частота процессора - количество исполняемых инструкций за единицу времени.
- Разрядность системы - количество обрабатываемой информации одной инстуркцией.
- Быстродействие памяти и системных устройств - уменьшение времени ожидания (простоя процессора):
	- Кеш-память.
		![[Pasted image 20260212124647.png]]
	- Сегментно-страничная организация памяти.
	- "Ленивое" распределение памяти.
- Аппаратная реализация базовых вычислительных алгоритмов - аппаратное ускорение выполнения фундаментальных операций.
- Гетерогенные вычислительные системы - разделение задач между специализированными процессорами (сопроцессорами, контроллерами).
- Паралеллизм вычислений:
	- Вычислительные кластеры.
	- Многопроцессорные и многоядерные системы.
	- Массивно-параллельные вычисления.
	- Векторные вычисления.
	- Распараллеливание конвейера.

Модели параллельных вычислений - классификация Флинна:
- SISD: Single Instruction - Single Data
- SIMD: Single Instruction - Multiply Data
- MISD: Multiply Instruction - Single Data
- MIMD: Multiply Instruction - Multiply Data

Архитектура процессоров:
- CISC - Complex Instruction Set Computer
- RISC - Reduced Instruction Set Computer
- MIPS - Microprocessor without Interlocked Pipeline Stages
- Микрокод - сикропрограммируемые процессоры

#### Ускорение программ
Коэффициент ускорения (ускорение программы при распараллеливании):
$$S=\frac{T_{seq}}{T_{par}}$$

Эффективность распараллеливания ($p$ - количество процессов/потоков):
$$E=\frac{S}{p}$$

Закон Амдала - показывает предельное (теоретическое) ускорение программы при **постоянном объеме работы**:
$$S_p=\frac{1}{\alpha+\frac{1 - \alpha}{p}}$$

Закон Густафсона - показывает предельное (теоретическое) увеличение объема работы при **постоянном времени**:
$$S_n=\alpha+(1-\alpha)*p$$
