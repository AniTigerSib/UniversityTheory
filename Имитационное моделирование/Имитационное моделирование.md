## Система
**Система** - это
1. *Описательное* - совокупность объектов, взаимодействующих между собой;
2. *Конструктивное 1* - средство достижения цели;
3. *Конструктивное 2* - отражение в сознании субъекта свойств объектов и их отношений в решении задач исследования познания.

### Этапы построения систем
1. Осознание проблемной ситуации;
2. Определение целей системы;
3. Определить функционал и свойства системы;
4. Определить структуру системы;

### Исследование систем
Существует **классический** и **системный** подход к исследованию систем:
- **Классический подход** - разбиение на составляющие.
- **Системный подход** - от общего к частному (организм).

#### Принципы системного подхода
1. Принцип дедуктивной последовательности (от общего к частному);
2. Принцип интегрированного рассмотрения. Система должна быть неразъемна как целое;
3. Принцип согласования ресурсов и целей рассмотрения;
4. Принцип бесконфликтности. Отсутствие конфликтов между частями целого.

#### Задачи в исследовании систем
Можно выделить 2 задачи:
1. **Синтеза системы**;
2. **Анализа системы**.

### Классификация систем
**Признаки** классификации:
- По *виду отображаемого объекта*:
	1. Материальные системы (технические, экономические, общественные и т.д.);
	2. Абстрактные системы (научные знания разного научного направления...).
- По *характеру протекаемых процеесов*:
	1. Детерминированные системы (результат зависит только от входных данных);
	2. Стохастические системы (присутствуют случайные воздействия).
- По *степени организованности*:
	1. Хорошо организованные системы;
	2. Плохо организованные системы;
	3. Самоорганизующиеся системы.
- По *сложности структуры и поведения*:
	1. Простые системы;
	2. Сложные системы.

#### Признаки сложной системы
- Многомерность системы;
- Многообразие возможных форм связи между элементами системы;
- Многокритериальность (от слова критерий);
- Многообразие природы элементов системы;
- Способность сохранять структуру;
- Многоплановость в научном направлении.

>[!tip] Необязательно удовлетворять всем криетриям

## Моделирование
> **Моделирование** - это вид исследования систем на их **моделях**.

>**Модель** - это объект-заменитель объекта-оригинала, который отображает важные черты оригинала с точки зрения исследователя для решения поставленных задач.

Модели должны быть подобны системам. Различают разные виды подобия.
#### Виды моделей
- *В основном*:
	1. Познавательные модели;
	2. Прагматические модели.
- По *протекаемому времени в модели*:
	1. Статические модели;
	2. Динамические модели.

>[!tip] Время может продвигаться непрерывно, может дискретно, а есть модели без учета времени в системе.

#### Виды подобия систем
- *Прямое подобие*:
	- Пространственное подобие;
	- Физическое подобие.
- *Косвенное подобие* - объективно существует в виде близости абстрактных моделей как объекта, так и самой системы;
- *Условное подобие* (условно-графические изображения и т.д.).

#### Свойства моделей
- Приближенность модели;
- Адекватность модели;
- Ингерентность (согласование с внешней средой) модели;

#### Принципы моделирования
- Принцип информационной достаточности;
- Прицнип осуществимости (временные рамки/границы, которые нас удовлетворяют);
- Принцип сножественности (одной модели, как правило, недостаточно для исследования всей систмы целиком);
- Принцип аггрегирования (разбиение системы на подсистемы/аггрегаты);
- Принцип параметризации.

### Подходы в имитационном моделировании
- Дискретно-событийный подход (процессный подход). Используется на нижнем уровне синхронизации;
- Агентное моделирование;
- Метод системной динамики. Самый верхний уровень абстрагирования;
- Узкоспециализированный подход.

### Этапы имитационного моделирования

1. Проблема + цель -> 
	- Содержательная модель
2. Разработка концептуальной модели ->
	- Концептуальная модель
	- Способ реализации модели
3. Формализация имитационной модели ->
	- Формальное описание
4. Программирование (настройка) имитационной модели ->
	- Имитационная модель
   Подготовка исходных данных (**на одном уровне с программированием**)
5. Планирование имитационных экспериментов
6. Проведение экспериментов
7. Интерпретация результатов экспериментов

*А после этого задаемся вопросом "Устраивает ли нас результат?".*

## Е-сети

Двудольный граф (нет параллельных связей-дуг, только одна).
$$E = (P, T, P_{pre}, P_{post}, \mu_0)$$
***
Существует 3 вида позиций:
1. $S$ Простая позиция (маркируемая). *Кружочек.* Количество фишек не более одной.
2. $Q$ Очередь (маркируемая). *Овал.* Количество фишек любое.
3. $R$ Не маркируется. *Квадратик.* Содержит условие срабатывания перехода. Стандартная разрешающая процедура - первая (по номеру дуги) сверху занятая входная / первая (по номеру дуги) сверху свободная выходная.
***
Функция предшествования отображает количество дуг идущих из позиции в переход.
Функция следования отображает количество дуг идущих из перехода в позицию.
***
Фишка снабжается свойствами (атрибутами).
***
Е-сеть представима набором элементарных сетей:
1. $T$ -> $Y(S, Q)$: генератор фишек. Условием для срабатывания будет являться отсутствие маркера в *простой* позиции (если очередь - пофиг). **ВАЖНО:** время активности генератора не должно быть равно 0.
2. $X(S, Q)$ ->$T$: поглотитель. Позволяет выводить объекты из системы. Условием для срабатывания будет являться наличие маркера в позиции.
3. $X(S, Q)$ -> $T$ -> $Y(S,Q)$: простой переход. Условием для срабатывания будет являться наличие маркера во входной позиции, а также его отсутсвие в *простой* выходной позиции.
4. ![[Pasted image 20251006103615.png]]Переключатель выходной. Условием срабатывания являются: занятая входная позиция и свободная $i$-ая позиция, где $i$ задается разрешающей процедурой, которая принимает значения $R(1,n)$.
5. ![[Pasted image 20251006104232.png]]Переключатель входной. Условием срабатывания являются: занятая входная $i$-ая позиция, где $i$ задается разрешающей процедурой, которая принимает значения $R(1,n)$, и свободная выходная позиция.
6. ![[Pasted image 20251006104426.png]]Размножитель. Условием срабатывания являются занятая входная позиция и все выходные (простые) свободные. Свойства фишки копируются в каждую выходную.
7. ![[Pasted image 20251006104915.png]]Соединитель. Условием срабатывания являются все занятые входные позиции,а также свободная (простая) выходная. Атрибуты скопируются из первой входной дуги.
8. ![[Pasted image 20251006105434.png]]Прерывающая элементарная сеть. Первая дуга отображается основное (первое) действие. Услвием срабатывания является отсутвие фишки в прерывающей входной позиции. После фишки на обоих - они перейдут в выходные после ожидания. После фишки только на прерывающей - она переходит во вторую выходную.
***

Каждый переход может быть:
- Пассивным - условия для срабатывания нет.
- Активным - есть условия для срабатывания и они выполняются в полном объеме.
- Взведенным - есть условия для срабатывания но они выполняются не полностью.

Время активности переходов стандартное равно 0 (кроме генератора).

### Среда моделирования EVA

#### Схема

![[Pasted image 20251015101436.png]]

Расширение файла графа сети: `.EEE`
Расширенирение файла текстового описания сети: `.DEF`
Именования файлов одинаковые
Основа языка описания - `EDE` (базирован на `C`)

#### Синтаксис

- Регистрозависимый язык
- Название разделов пишется с большой буквы
- В каждом разделе метка перехода устанавливается (не упоминается, а устанавливается ($T_i$)) единожды
##### Раздел A

Раздел объявления процедур, функций и т.д.
A: ```int _a;```

Предпочтительно `_` перед идентификатором пользовательских переменных.
Также предпочтительно не использовать идентификаторы, сходные с системными.

##### Раздел I

Инициализируются переменные
I: ```_a = 10;```

##### Раздел R

Описание разрешающей позиции
R: $T_i:$ ```r = 0;```

##### Раздел D

Назначается время активности перехода
D: $T_i:$ ```d = 5;```

##### Раздел T

Предполагает преобразование переменных и атрибутов
T: $T_i:$ ```_a++;```

##### Раздел S

Содержит функции статистики, лексика не Сишная

##### Раздел E

Раздел выполнения процедур по окончанию заданного периода моделирования

#### Системные переменные

- `d` - время активности перехода
- `r` - значение разрешающей позиции
- `Clock` - текущее время
- `Finish` - конечное время
- `s[i].atr[j]` - обращение к атрибуту
	- Пример: `s[1].atr[1]` - первый атрибут фишки, находящийся в первой простой позиции
- `q[i].first->atr[j]` или `q[i].last->atr[j]` - обращение к атрибуту фишки в очереди
	- Возможно обращение только к первой либо последней фишке в очереди
- `s[i].nmark` - 0 или 1
- `q[i].strategy` - F или L. Назначает стретегию выхода из очереди (FIFO или LIFO, по умолчанию **FIFO**)
- Генерация случайных чисел только по равномерному закону, либо экспоненциальному закону.
	- `rnd(N)` - равномерный закон `[0, N)`
	- `expon(N)` - экспоненциальный закон
- `#include "Userfunc.h"` - заголовочный файл библиотеки дополнительных функций языка
	- `fsort(Nq,Na,Z)` - сортировка очереди по атрибуту
		- `Nq` - номер очереди (`int`)
		- `Na` - номер атрибута (`int`)
		- `Z` - направление сортировки (`long`, приписка L)
	- `ffix(Nq,Na,Matr)` - поиск фишки с конкретным значением атрибута и установка ее в начало очереди
		- `Nq` - номер очереди (`int`)
		- `Na` - номер атрибута (`int`)
		- `Matr` - значение атрибута

#### Ститистические функции (раздел S)

- `stat_s(Ns,Min,St,Nst)` - статистика перехода
	- `Ns` - номер позиции (`int`)
	- `Min` - левая граница гистограммы (`long)
	- `St` - шаг (`long`)
	- `Nst` - количество столбцов гистограммы (`int`)
	- Returning:
		- Суммарное время занятости простой позиции
		- Данные гистограммы времени занятости простой позиции
- `stat_a(Ns,Na,Min,St,Nst)` - статистика атрибута (только для простой позиции)
	- `Ns` - номер позиции (`int`)
	- `Na` - номер атрибута (`int`)
	- `Min`, `St`, `Nst` - данные для гистограммы (аналогично выше)
	- Returning:
		- Только данные гистограммы значения атрибута
- `stat_q(Nq,Min,St,Nst)` - статистика очереди
	- `Nq` - номер очереди (`int`)
	- `Min`, `St`, `Nst` - данные для гистограммы (аналогично выше)
	- Returning:
		- Максимальная длина очереди
		- Интегральное время занятости очереди (по отношению к каждой фишке)
		- Данные для гистограммы времени занятости очереди
- `stat_t(Nt,Min,St,Nst)` - статистика перехода
	- `Nt` - номер перехода (`int`)
	- `Min`, `St`, `Nst` - данные для гистограммы (аналогично выше)
	- Returning:
		- Количество пройденных фишек через переход
		- Суммарное время активности перехода
		- Данные для гистограммы времени активности перехода

#### Модели дисциплин обслуживания

Обслуживание может быть:
- Бесприоритетное
- С приоритетами

##### Бесприоритетное обслуживание

Каждый генератор воспроизводит сущности с разными приоритетами. Чем выше генератор - тем выше приоритет.
Для обслуживания предоставляется константная величина времени. Если обслуживание не заершилось за это время - задание уходит на доработку.

##### Приоритетное обслуживание

- Абсолютный приоритет - прерывания
- Относительный приоритет - без прерываний, но с соблюдением приоритетов

## Постановка целемоделирования

$$W = W(q, c, t)$$
Где:
$q$ - полезный эффект
$c$ - стоимостные характеристики
$t$ - временные затраты

*Могут быть представлены векторами.*

$W = \sum_{j}^{k}(W_i*q_i)$

Когда показатели эффективности разнородные - частные показатели нормируют.
